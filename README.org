#+title: flymake-janet
#+author: Gal Buki
#+language: en

A Flymake backend for the [[https://janet-lang.org][Janet]] programming language. \\
It uses Janet's built-in linter and provides compile errors and lint warnings to Flymake.

* Requirements

- Emacs 26.1 or later
- The =janet= binary on your =PATH=
- Either [[https://github.com/ALSchwalm/janet-mode][janet-mode]] or [[https://github.com/sogaiu/janet-ts-mode][janet-ts-mode]]

* Installation

** Manual

Clone the repository and add it to your load path:

#+begin_src emacs-lisp
  (add-to-list 'load-path "/path/to/flymake-janet")
  (require 'flymake-janet)
#+end_src

** use-package with :vc (Emacs 29+)

#+begin_src emacs-lisp
  (use-package flymake-janet
    :vc (:url "https://github.com/torusjkl/flymake-janet")
    :custom
    (flymake-janet-warn-level 'strict)
    (flymake-janet-error-level 'relaxed)
    :hook
    ((janet-mode janet-ts-mode) . flymake-mode)
    ((janet-mode janet-ts-mode) . flymake-janet-setup))
#+end_src

* Setup

Call =flymake-janet-setup= in the buffer where you want the backend active. \\
The typical approach is to add it to =janet-mode-hook=:

#+begin_src emacs-lisp
  (add-hook 'janet-mode-hook #'flymake-janet-setup)
#+end_src

This adds the backend and starts Flymake if it is already running. \\
You still need to enable Flymake itself — either globally or per hook:

#+begin_src emacs-lisp
  (add-hook 'janet-mode-hook #'flymake-mode)
#+end_src

** Using with Eglot

If you use Eglot with a Janet LSP server, Eglot takes over Flymake by default and will disable other backends. \\
To keep flymake-janet running alongside Eglot, tell Eglot to stay out of Flymake and activate the backend via =eglot-managed-mode-hook=:

#+begin_src emacs-lisp
  (add-to-list 'eglot-stay-out-of 'flymake)
  (add-hook 'eglot-managed-mode-hook #'flymake-janet-setup)
#+end_src

Note that =eglot-stay-out-of= is a global list. \\
It affects all languages managed by Eglot, not just Janet. To undo this, remove the entry explicitly:

#+begin_src emacs-lisp
  (setq eglot-stay-out-of (delete 'flymake eglot-stay-out-of))
#+end_src

* Settings

** Warn level

Controls the lint level passed to =-w LEVEL=.
Warnings at or below this level are reported. \\
Default is =normal=.

| Value      | Meaning                          |
|------------+----------------------------------|
| ='relaxed= | Only the most important warnings |
| ='normal=  | Standard warnings (default)      |
| ='strict=  | All warnings, including pedantic |
| =nil=      | Omit the =-w= flag entirely      |

#+begin_src emacs-lisp
  (setopt flymake-janet-warn-level 'strict)
#+end_src

** Error level

Controls the lint level passed to =-x LEVEL=.
Warnings at or below this level are promoted to errors. \\
Default is =nil= (disabled).

| Value      | Meaning                                  |
|------------+------------------------------------------|
| ='relaxed= | Promote only the most important warnings |
| ='normal=  | Promote standard warnings                |
| ='strict=  | Promote all warnings                     |
| =nil=      | Omit the =-x= flag entirely (default)    |

#+begin_src emacs-lisp
  (setopt flymake-janet-error-level 'strict)
#+end_src

** Message format

By default, the =compile error:=, =compile warning:=, and =parse error:= prefixes are stripped from diagnostic messages,
since the severity is already indicated by Flymake. \\
The regex can be changed by setting =flymake-janet-message-strip-regexp=. \\
Set it to =nil= to show the full message text.

#+begin_src emacs-lisp
  (setopt flymake-janet-message-strip-regexp nil)
#+end_src

* How it works

On every buffer change, flymake-janet runs a bundled Janet script (=flymake-janet-compile-all.janet=) that reads the buffer from stdin
and uses Janet's =run-context= with dedicated callbacks to collect parse errors, compile errors, and lint warnings in a single pass. \\
Definition forms (=defn=, =def=, =defmacro=, ...) are executed so that later forms in the file can reference them. \\
All other forms are skipped to avoid side effects.
